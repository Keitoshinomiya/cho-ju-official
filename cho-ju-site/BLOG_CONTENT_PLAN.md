# CHO-JU ブログ コンテンツプラン（100本）

collegrance のSEO量産方式を参考にした、CHO-JU Journal の記事企画リスト。

## 方針
- **1カテゴリ＝検索ニーズのかたまり。** 各カテゴリを10〜18本で面的に攻める。
- **タイトルに必ず「悩み・シーン・キーワード」を入れる。** （例:「夜中のトイレが怖い」「消しカス 床に散らからない」）
- **各記事に関連商品を紐づけ**、記事末CTA（`productId`）で購入導線へ。
- **執筆は型に沿う**：導入（共感）→ 悩みの言語化 → 解決のヒント → 根拠/使い方 → まとめ → CTA。
- **量産コマンド**：`npm run new:post -- --slug "..." --title "..." --category "..." --product "..." --excerpt "..." --tags "a,b,c"`
- カテゴリ定義は `lib/categories.ts`、ひな形は `scripts/new-post.mjs`。

## 進捗の見方
- [x] = 執筆済み（`posts/` に存在） / [ ] = 未着手（プラン段階）

## 画像について
- 現状の記事画像は `public/images/blog/` に6枚。100本展開には記事ごとの画像が必要。
- 当面は商品カテゴリ別の既定画像（`scripts/new-post.mjs` の `defaultImageByProduct`）で代用可能。
- 本番ではカテゴリ別に10〜20枚のストック画像を用意し、frontmatter の `image:` を差し替える運用を推奨。
